🔧 Test - Correction Erreur Cyclique Exportateurs

📋 Description du Test

Ce test vérifie que la correction de l'erreur "cyclic object value" fonctionne correctement lors de l'utilisation du filtre avancé dans l'écran "Exportateurs Agrées".

Problème résolu : L'objet filters passé dans la query key contenait des références circulaires qui ne pouvaient pas être sérialisées par React Query.

🧪 Tests à Effectuer

  1. Test 1 : Accéder à la page Exportateurs Agrées
  2. Test 2 : Utiliser le filtre avancé avec différents critères
  3. Test 3 : Vérifier qu'aucune erreur "cyclic object value" n'apparaît
  4. Test 4 : Tester la pagination avec des filtres actifs

✅ Corrections Appliquées

  • Fonction de sérialisation sûre : Ajout de safeSerializeFilters() dans useExporters.ts
  • Query key sécurisée : Utilisation de safeSerializeFilters(filters) au lieu de l'objet filters directement
  • Prévention des références circulaires : La fonction ne sérialise que les valeurs primitives et nettoie les valeurs undefined/null

🎯 Résultat Attendu

Après la correction, l'utilisation du filtre avancé ne devrait plus générer l'erreur "Uncaught TypeError: cyclic object value" et les filtres devraient fonctionner correctement.
